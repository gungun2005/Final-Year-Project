How it Works

The technical architecture behind our AI-driven diagnosis system.

Step 1: Frontend & Input

The user uploads a leaf image via our FastAPI-hosted web interface. JavaScript handles the image preview and asynchronous API communication.

Step 2: Universal Pre-Scanner

We use the Google Gemini 2.0 Flash model to perform a safety check. It ensures the image is actually a plant and detects if it is universally healthy before sending it to our specialized model.

Step 3: CNN Disease Identification

The image is processed by a Convolutional Neural Network (CNN) built with TensorFlow/Keras. It uses a trained .h5 model to identify specific diseases based on pixel patterns.

Step 4: Generative Treatment Planning

Once diagnosed, the result is sent back to Gemini with strict Expert Prompt Engineering. This generates a localized, actionable treatment plan in the user's chosen language.

Tech Stack Summary

  • FastAPI (Python Backend)
  • TensorFlow / Keras (CNN)
  • Google Generative AI (NLP)
  • Tailwind CSS (UI Framework)
  • Web Speech API (TTS/STT)
